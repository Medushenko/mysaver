# 🔍 2. Анализ рынка и UX-принципы

## 🥊 Прямые конкуренты
| Продукт | Плюсы | Минусы | Как мы отличаемся |
|---------|-------|--------|------------------|
| MultCloud | 30+ провайдеров, server-side copy, планировщик | Нет десктопа, лимиты на Free, базовый UI | Гибкий парсер задач, отчёты с retry, мобильный PWA + бот |
| cloudHQ | Бесплатный трафик, интеграции с почтой/Slack | Файловый менеджер рудиментарный, >100МБ платно | Прозрачные конфликты, квоты, теги, смарт-коллекции |
| rclone | 70+ бэкендов, CLI, open-source | Нет GUI, высокий порог входа, нет отчётов/UI | Готовый веб-интерфейс, визуализация, парсер, бот |

| Продукт | Паттерн | Применение в MySaver |
|---------|---------|---------------------|
| Zapier/Make | «Link & Done», визуальные пайплайны | Авто-распознавание ссылок → превью задачи → 1 клик |

| Telegram Bots | Inline-кнопки, push-статусы | `/save`, `/status`, `/retry` без входа в веб |

## 💢 Боли пользователей (на основе отзывов)
- «У меня 5 облаков, не помню, где что лежит» → Единая карта + кросс-поиск
- «Перенос = скачать/загрузить, долго» → Server-side copy через адаптеры
- «Боюсь потерять файлы, нет контроля» → Пост-верификация + отчёт с ссылками
- «Нужно найти файл по контексту, не по имени» → ИИ-поиск по метаданным (V2)
- «Хочу управлять с телефона» → PWA + бот + мобильный UX

## 🎨 Ключевые UX-принципы
1. **Прогрессивное раскрытие**: базовый ввод → «⚙️ Дополнительно» → опции.
2. **Конфликт-UI**: side-by-side метаданные, кнопки `⏭ Пропустить`, `🔄 Перезаписать`, `⏰ Позже`.
3. **Статусная прозрачность**: очередь → выполняется → готово/ошибка → лог → retry.
4. **Mobile-First**: свайпы, bottom-sheet, микрофон для голоса, виртуализация списков.
5. **Безопасный гость**: сессия → работа → авто-очистка, без следов.
