# 🧩 5. Функциональная спецификация

## 📥 5.1 Парсер задач
- Поддержка: TXT, MD, YAML/JSON (MVP), PDF/DOCX (V2)
- Логика: извлечение URL → определение провайдера → построение дерева
- Интерактивное превью: чекбоксы, массовое снятие, фильтры по расширению/размеру
- Прямой UI-поток: выбор в интерфейсе → валидация → очередь (без дерева)

## 🔍 5.2 Верификация и конфликты
- По умолчанию: `rclone --checksum` (MD5/SHA1/provider-hash)
- Конфликты: `skip_identical`, `overwrite_newer`, `keep_both`, `resolve_later`
- Хранение решений: БД + UI-история + отчёт
- Fallback: если хеш недоступен → `size + mtime`

## 📏 5.3 Контроль квот и роутинг
- Предварительный расчёт: асинхронный `scan-folder` → сравнение с кэшем квоты
- Режимы: `✅ Check before start`, `🔄 Live check`, `❌ Skip`
- Fallback: при `QUOTA_EXCEEDED` → предложение перенаправить остаток
- Отчёт: блок «Не помещено» + кнопка «Выбрать другой приёмник»

## 📊 5.4 Отчётность и Retry
- Формат: JSON → HTML (UI) / PDF (export)
- Структура: сводка, успешные, конфликты, ошибки с ссылками, принятые решения
- Retry: фильтрация `error_paths` → авто-создание подзадачи
- Логирование: `OFF` / `BASIC` / `VERBOSE` (влияет на время выполнения)

## 🔐 5.5 Сессии и доступ
- Гость: TTL 24ч, авто-очистка токенов, `usage_logs` привязаны к `session_id`
- Регистрация: сохранение OAuth2 токенов (шифрование в БД), `is_pro` флаг
- Роли MVP: `Owner` (полный доступ), `Viewer` (только чтение/отчёты)

## 🚧 Границы MVP
- ✅ Ядро копирования, парсер, превью-дерево, отчёты, бот, квоты
- ❌ Шифрование, ABAC, десктоп-агент, AI-поиск по контенту, биллинг
