# 🗺 8. Дорожная карта и итерации

## 🟢 Фаза 0: Подготовка (1–2 недели)
| Итерация | Задача | Результат |
|----------|--------|-----------|
| 0.1 | Репозиторий, CI/CD, структура docs | `mysaver/` готов к работе |
| 0.2 | Прототип БД + очередь | `users`, `tasks`, `usage_logs`, Redis + Celery |
| 0.3 | UI-макет (Figma → HTML/React) | Утверждённый mobile-first интерфейс |

## 🟡 Фаза 1: MVP Ядро (4–6 недель)
| Итерация | Задача | Критерий приёмки |
|----------|--------|------------------|
| 1.1 | Адаптер Яндекс.Диск + rclone RC | 1 файл: локально → Яндекс, статус OK |
| 1.2 | Адаптер Google Drive + парсер ссылок | 3 ссылки → распознано 2 провайдера |
| 1.3 | Очередь + воркер + статусы | Задача в фоне, не блокирует UI |
| 1.4 | Веб-дашборд + WebSocket прогресс | Real-time полоса, скорость, ETA |
| 1.5 | Отчёты + кнопка «Повторить ошибки» | PDF/HTML, retry только failed |
| 1.6 | Telegram-бот (базовый) | `/save`, `/status`, inline-кнопки |
| 1.7 | Гостевой режим + TTL-очистка | Токены удалены, сессия закрыта |

## 🔵 Фаза 2: Расширение (6–8 недель)
| Итерация | Фокус | Когда привлекать подрядчика |
|----------|-------|---------------------------|
| 2.1 | Dropbox, S3, локальный диск | При багах в rate-limit/multipart |
| 2.2 | Теги + наследование | Frontend-специалист (React/VirtualList) |
| 2.3 | Смарт-папки | Backend-архитектор (query optimizer) |
| 2.4 | Контроль квот + fallback | DevOps/SRE (monitoring setup) |
| 2.5 | Простые роли + аудит | Security-консультант (pen-test) |

## 🟣 Фаза 3: Масштабирование (по запросу)
- AI-поиск по метаданным, голосовой интерфейс
- Десктопный агент (macOS/Win), on-demand загрузка
- Биллинг, SSO, SCIM, Enterprise-роли

## 📊 Метрики успеха MVP
- `Time-to-first-task` < 45 сек
- `Task completion rate` > 92%
- `Error retry success` > 85%
- `Mobile UX satisfaction` > 4.2/5
